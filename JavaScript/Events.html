Events in js

Event handling allows us to execute javascript code in response to user actions or browser events.

Types of Events

  • mouse events : click, dblclick, mouse over, mouse down, mouse up...etc
  • keyboard events : keydown, keyup, keypress.
  • form events : submit, focus, input...etc
  • window events : scroll, resize, load, unload...etc

Here two ways to handle the events

  • 1. Inline event handlers
  • 2. Event listeners

1. Inline event handlers

Drawback of inline event handlers:
1.we cannot attach multiple event listeners to a single element
2.it mixes both HTML and JS code.

2.addEventListener()

•This method provides flexible way to handle events
•we can attach same events multiple times to single events.

syntax : element_name.addEventListener("event",function); 
        
        let button = document.getElementById("btn");
        button.addEventListener("click", ()=>{
            console.log("This is task A");
        });
        button.addEventListener("click", ()=>{
            console.log("This is task B");
        });

Example for keyboard events(keypress,keydown,keyup)

  • keydown : Fires when a key is pressed down. fires for all keys like letters,ctrl,alt,shift,arrows
  • keypress : Fires when a key is pressed and released. works only for pritable charaters and doesn't detect key like ctrl,alt,shift,arrow
  • keyup : Fires when a key is released. fires once after release, good for final input detection.
        
        let user = document.getElementById("user")
        user.addEventListener("keydown", (e) => {
            console.log(e.key);
        });
        // code for keyup
        let user = document.getElementById("user")
        user.addEventListener("keyup", (e)=>{
            console.log(e.key);
        });
        // code for keypress
        let user = document.getElementById("user")
        user.addEventListener("keypress", (e)=>{
            console.log(e.key);
        });
simple key movement code :
        
        let user1= document.getElementById("user1")
        user1.addEventListener("keydown", (e) => {
            if (e.key=="a"){
                console.log("move left")
            }else if (e.key=="d"){
                console.log("move right")
            }else if (e.key=="w"){
                console.log("move up")
            }else if (e.key=="s"){
                console.log("move down")
            }else{
                console.log("invalid key")
            }
        });

Event bubbling :

• event bubbling is adefault behaviour in DOM, where an event starts from targeted element and propagates through its parent elemt
• to stop event bubbling we use .stopPropagation() Method.

  1. event capturing(trickling) : Top-To-Down process
    Document -> Grandparent -> Parent -> child

  2. event bubbling : Button-Top-Top process
    child -> parent -> grandparent -> document
        
        //event bubbling
        let parent = document.getElementById("parent")
        let child = document.getElementById("child")

        parent.addEventListener("click", () => {
            confirm("parent click");
        })

        child.addEventListener("click", (e) => {
            e.stopPropagation() //to stop bubbing 
            confirm("child click");
        })
